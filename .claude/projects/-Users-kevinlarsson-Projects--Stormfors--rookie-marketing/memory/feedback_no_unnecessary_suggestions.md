Only suggest fixes and improvements that are actually needed. Don't pad audit results or outstanding lists with speculative nice-to-haves (e.g. cacheLife, output: standalone, CSS animation replacements) that don't solve a real problem for this project.

**Why:** User called this out directly — suggesting unnecessary work erodes trust and wastes time reviewing things that don't matter.

**How to apply:** Before recommending something, ask: "Is this solving a real problem in this project right now?" If the answer is "maybe later" or "in theory," don't include it.
